## このサイトもClaude Codeで作っている

最初に正直に言うと、いまあなたが読んでいるこの「AI仕事術」というサイトも、Claude Codeで構築しました。Next.jsプロジェクトの初期化、ヘッダー・フッター・記事ページのコンポーネント設計、Markdown記事のパース、OGP画像の動的生成、VercelデプロイとDNS設定、すべてClaude Codeとの対話だけで進めています。私自身は建築設計が本業で、フロントエンドの専門家ではありません。それでも14記事を持つアフィリエイトサイトを2日で立ち上げられたのは、Claude Codeのおかげです。

この記事はその過程で気づいたこと、つまずいたこと、回避策、そして実際に発生したAPI料金まで、本音で書きます。検索すると出てくる「Claude Codeはすごい」だけの紹介記事には書かれていないことを書きます。

## Claude Codeとは何が違うのか

Claude Codeを一言で言うと、「Anthropicが作った、ターミナル上で動くAIコーディングエージェント」です。ChatGPTやGitHub Copilotとは決定的に違う点が3つあります。

ひとつめは、**プロジェクト内のファイルを自分で読み書きする**こと。ChatGPTにコードを質問するときは、コードをコピーして貼り付ける必要があります。Copilotは現在のファイルしか見えません。Claude Codeは「src/components配下の構造を理解してから、新しいコンポーネントを既存のスタイルに合わせて追加して」と頼めば、自分でファイルを読み、構造を把握し、規約に従って新規ファイルを作ります。

ふたつめは、**シェルコマンドを実行できる**こと。`npm install`、`git commit`、`npm run build`、`vercel deploy`まで、ターミナル上の操作はすべて任せられます。私がこのサイトを構築したときも、`gh repo create`でGitHubリポジトリを作る、`vercel env add`で環境変数を追加する、といった操作を全部Claude Codeに任せました。

みっつめは、**プランを立てて段階的に実行する**こと。「ブログサイトを作って」のような大きな指示に対して、Claude Codeは内部でTodoリストを作り、ひとつずつ実行しながら進捗を報告してくれます。途中でユーザーが横やりを入れて方針を変えても、追従してくれます。

この3つの組み合わせが、これまでのAIコーディング支援とは別物の体験を作ります。

## 半年使った実際の作業ログ

私がClaude Codeを使い始めたのは2025年10月。当初はAnthropicの内部ツールに近い扱いで、コマンドラインに馴染みのある人だけが触っていました。半年で以下のプロジェクトをClaude Codeで構築しました。

ひとつめは、設計事務所の社内向け勤怠管理アプリ。Next.js + Supabaseで、打刻、シフト管理、有給管理、CSV出力までついた本格的なアプリです。フルスクラッチで作って所要時間は丸4日。同じ規模のアプリを以前React + Firebaseで作ったときは2週間以上かかったので、3〜4倍速くなった感覚です。

ふたつめは、X（Twitter）の自動投稿Bot。OpenAI APIで投稿候補を生成し、Twitter API v2で予約投稿するNode.jsアプリ。ライブラリの選定からデプロイまで含めて1日。

みっつめが、いまあなたが見ているこのサイト「AI仕事術」。Next.js + Tailwind + Markdownで、ドメイン取得、Vercelデプロイ、Google Analytics連携、Google Search Console認証、A8.netのアフィリエイトリンク埋め込み、すべて含めて2日。記事執筆もClaude Codeにドラフトを書かせ、私が体験ベースで書き直しています（この記事もそうです）。

通算して半年でだいたい200時間くらいClaude Codeを動かしました。その間に発生した実費が次の話です。

## API料金は実際いくらかかったか

ここがClaude Code紹介記事でいちばん隠されている部分です。「すごい」だけの記事は料金には触れません。

私のAnthropicの請求書を確認すると、Claude Codeを真剣に使い始めた2025年10月から2026年3月までの6ヶ月で、合計約**420ドル**を支払っています。月平均にすると約70ドル、日本円で約1万500円。

月別の内訳はバラついていて、軽い月（既存サイトの修正だけ）は月30ドルくらい、重い月（新規プロジェクトを立ち上げて毎日数時間動かす月）は月150ドルくらいになります。「AI仕事術」を構築した今月は、立ち上げ作業で2日連続で動かしたので、いまのところ月60ドルくらいのペースです。

これを高いと見るか安いと見るかは、その時間で作った成果物の価値次第です。私の場合、200時間相当の作業を実質50〜60時間で終わらせている感覚なので、削減できた140時間を時給5,000円で計算すると70万円分の価値。月420ドル（約63,000円）の投資としては、10倍以上のリターンが出ています。

ちなみに、毎日Claude Codeを使わない人は、AnthropicのClaude Pro（月20ドル）やMax（月100ドル）プランの方が安くなる場合もあります。これは**APIの従量課金とは別の定額プラン**で、Claude Codeを定額枠内で使えます。重めに使う私のような人は、いまMaxプランに移行中です。

## 1日の典型的なワークフロー

朝9時にClaude Codeを起動して、その日の作業を伝えます。

```
今日やること:
1. 昨日のシフト管理アプリで残ったバグを直す
2. 経費アプリのOCR機能をfreeeに連携できるようにする
3. このサイトに新しい記事を1本追加する
```

Claude Codeは内部でTodoリストを作り、優先順位を確認してきます。私が「1から順番でいい」と返すと、まず1の調査を始めます。

「昨日のバグ」と一言伝えただけで、Claude Codeは git log を確認して昨日の変更を見つけ、それに関連するエラーを再現するためのコマンドを実行します。エラーメッセージを読んで原因を特定し、修正案を提示してきます。私が「いいよ」と返すと、ファイルを書き換え、`npm run build`でビルドを通し、`git commit`まで進めます。私の作業は「いいよ」と言うことだけです。

2と3も同じ流れで進みます。途中で私の判断が必要な場面では、Claude Codeから質問が飛んできます。「freeeのAPIに連携する方法は3通りあるが、どれにする？」と聞かれたら、それぞれの違いを聞き返して、決定します。

夕方17時にいったん区切るときには、その日やったことのサマリを書き出してもらい、明日のTodoとして残してもらいます。これがそのまま次の日の作業の起点になります。

## つまずきポイントと回避策

ここから本題です。半年使うと、Claude Codeの「ハマりポイント」も見えてきます。検索すると出てくる紹介記事には書かれていない部分です。

### つまずき1：大きすぎるタスクで迷子になる

「ECサイトを丸ごと作って」のような曖昧で巨大なタスクを投げると、Claude Codeは進めようとして途中で迷子になります。たとえばユーザー認証の途中でカート機能を考え始めて、結局どちらも中途半端、ということが起きます。

**回避策**：必ず中粒度に分割して投げる。「まずユーザー登録のフォームだけ作って、動作確認できたら次に進める」のように区切ります。Claude Codeも内部でTodoリストを作りますが、こちら側でも明示的に区切る方が確実です。

### つまずき2：勝手に方針を変えてしまう

長い対話の途中でClaude Codeが「やっぱりこの方が良いと思います」と勝手に方針を変えることがあります。たとえば最初は「Tailwindで書く」と決めていたのに、途中から「CSS Modulesを使ってもいいですか」と提案してきて、流れで承認するとプロジェクトの一貫性が崩れます。

**回避策**：プロジェクトのルートに `CLAUDE.md` ファイルを置いて、技術スタックや絶対ルールを書いておきます。Claude Codeは起動時に自動でこのファイルを読むので、「Tailwindのみ。CSS Modulesは禁止」と書いておけば、勝手な方針転換を抑制できます。

### つまずき3：ファイルを消されかける

あまり頻繁ではありませんが、リファクタリング中に「もう使われていないので削除します」と言って、実は別の場所から呼ばれているファイルを消そうとすることがあります。一度本番環境のreact-hook-formの設定ファイルを消されかけて、ぎりぎりで気づきました。

**回避策**：Git作業をこまめにする。Claude Code自身に `git status` と `git diff` を実行させてから commit する習慣をつけます。確認なしに `git add -A && git commit` を任せず、変更内容のサマリを必ず読んでから承認します。あと、削除コマンドは反射的にYESを押さない。

### つまずき4：API料金が膨らむ

長文のドキュメントを何度も読み込ませると、Anthropicの請求がじわじわ上がります。ある月は調子に乗って大量のJSONログを解析させて、その月の請求が一気に120ドルを超えました。

**回避策**：プロンプトキャッシュを意識する。同じシステムプロンプトを使い続けると、2回目以降は90%オフになります。また、巨大なファイルは事前に手で抽出してから渡す方が安い場合があります。Claude Codeは便利ですが、なんでも投げれば最適化してくれるわけではありません。

### つまずき5：ローカル環境特有のエラーに弱い

私の環境は Windows 11 + Git Bashという少し変わった構成で、Claude Codeが「Linuxシェルのつもり」でコマンドを書いてくることがあります。`/dev/null` をそのまま使ってエラーになる、`mkdir -p` がうまく動かない、といった小さなトラブルが起きます。

**回避策**：CLAUDE.mdに環境情報を書いておきます。「Windows 11 / Git Bash環境。/dev/null はNUL、パスは / でなく \ が必要な場面あり」と書いておくと、Claude Codeが自動で配慮してくれます。

## 他のAIコーディングツールとの比較

私はGitHub Copilot、Cursor、ChatGPTのCanvas機能、すべて同時期に試しました。それぞれ得意分野が違います。

**GitHub Copilot**は、IDE内で「次に書きたいコード」を予測してくれるツールです。エディタを離れずに済むので、すでに頭の中に設計があるときに最強です。ただし、設計から考えてもらいたいときや、複数ファイルにまたがる修正をしたいときには弱い。

**Cursor**は、VS Codeをフォークしたエディタで、AIとの対話とコード編集を統合した体験を提供します。ファイル横断的な編集がGUI上で快適にできます。ただし、シェルコマンドの実行やGit操作までは任せきれない。

**ChatGPT Canvas**は、ブラウザ上でコード編集ができる機能です。手軽ですが、プロジェクト全体を読ませることはできず、一発勝負のスニペット作成向け。

**Claude Code**は、これらと違って「ターミナルで動く自律エージェント」というポジションです。エディタを開かずに、ターミナルだけで完結できる。シェルコマンドもGitもデプロイも任せられる。半面、IDE内のリアルタイム補完はないので、Copilotと併用するのが私のスタイルです。

実際の使い分けは、設計から実装まで任せたい大きな仕事はClaude Code、コードを書きながらの細かい補完はCopilot、というハイブリッドに落ち着いています。

## こんな人にはおすすめしない

最後に、Claude Codeを「すすめない人」についても書きます。記事のほとんどは「使うべき」と書いていますが、合わない人もいます。

ターミナル操作にアレルギーがある人には向きません。Claude Codeはあくまでターミナル上のツールなので、コマンドラインに恐怖心がある人にはハードルが高いです。そういう人にはCursorの方が良いです。

レガシーな大規模プロジェクトに突っ込むのも危険です。10年前のJavaコードベース100万行みたいな環境では、Claude Codeも文脈を掴みきれず、誤った修正を提案してきます。新規プロジェクトや、ある程度モダンなTypeScript/Pythonプロジェクトで真価を発揮します。

完全な初心者、つまり「変数とは何か」もわからない人にも、いきなりClaude Codeはおすすめできません。出てきたコードの正しさを判断できないと、Claude Codeが間違った提案をしたときに気づけないからです。

## まとめ：半年使った正直な感想

Claude Codeは、私の働き方を根本から変えました。以前なら「これは技術的にしんどいから諦める」と思っていた個人プロジェクトを、次々と立ち上げられるようになりました。シフト管理アプリ、Bot、このサイト、すべて2025年以前の自分なら手をつけなかったものです。

一方で、無敵のツールではありません。月70ドルの請求は決して安くないし、勝手に方針を変えるクセや、ファイルを消されかけるリスクもあります。CLAUDE.mdで縛り、こまめなGit確認、適切な分割をしないと、振り回されます。

これから始める人へのアドバイスは、**最初の1ヶ月は小さなプロジェクトで遊ぶ**ことです。本番のクライアント案件にいきなり投入せず、自分用のメモアプリやTodoツールを作って、Claude Codeのクセを掴んでください。1ヶ月後には、もう手放せなくなっているはずです。

そしてその時には、月数万円の請求書を見ても「安い投資だ」と思えるようになっています。
